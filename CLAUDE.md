### Reid's End-to-End Invoice Flow
1. **Detect** invoice email in any distribution list inbox
2. **Download** PDF attachment — or click link if no attachment (utilities often send links)
3. **Upload** to correct SharePoint folder (vendor vs. utility)
4. **Process** via invoice-processing skill (extract data, match vendor/property/GL, dedupe)
5. **Enter** bill into Propertyware via API (attach PDF)
6. **Move** PDF from active → completed folder
7. **Log** to processed_invoice_ledger.xlsx
8. **Auto-forward** misrouted invoices to accounting@ for canonical record

### Critical Rules
- **NEVER ENTER NEGATIVE AMOUNTS**: Bill amount = invoice face value. ALWAYS. Never apply R&M markup math to the entry amount. PW calculates markup automatically.
- **MISSING INVOICE #**: Generate fallback ref: `NOINV-MMDDYYYY-Amount` (vendor), `AcctStmt-Last4` (utility), `VendorCode-Date` (recurring). Never leave Ref No blank.
- **DUPLICATE PREVENTION**: Vendors send invoices multiple times. Before ANY entry, check: (1) processed_invoice_ledger.xlsx, (2) Propertyware pw_list_bills by vendor+invoice#+amount, (3) SharePoint folder for same filename. If match → DUPLICATE, do NOT enter.
- **WORK ORDER LINKING**: Always attach bill to matching WO. WO can be open or closed. Exceeding $250 NTE is fine for entry (NTE governs work approval, not bill entry).
- **PDF ATTACHMENT**: Every PW bill must have the invoice PDF attached. No exceptions. API cannot attach files — Reid uses browser automation (Cowork/Machine 2) to attach PDFs via PW UI after API bill creation.
- **Utility auto-emails** go to accounting@ distribution list.
